// Core Git Workflow

From Zero to Commit

git init

Initialize

Set up a new local repository or clone an existing one from GitHub. This creates the .git folder that tracks everything.

git add

Stage Changes

Move modified files to the staging area. Only staged changes will be included in the next commit. Use git add . for all files.

git commit

Commit

Save a snapshot of staged changes with a descriptive message. Commits are permanent checkpoints in your project's history.

git push

Push

Upload your local commits to the remote repository on GitHub so teammates can see and collaborate on your work.

git pull

Pull

Fetch and merge the latest changes from the remote into your local branch. Always pull before starting new work.

git merge

Merge & PR

Integrate branches back together. On GitHub, use Pull Requests for code review before merging into the main branch.

// Quick Reference

Essential Commands

Setup & Config

git config --global user.name
Set your name for all commits
git config --global user.email
Set your email for all commits
git init
Initialize a new local repository
git clone <url>
Clone a remote repository locally

Staging & Commits

git status
Show working tree and staged changes
git add <file>
Stage a specific file
git add .
Stage all changes in current directory
git commit -m "msg"
Commit staged changes with a message
git log --oneline
View compact commit history

Branching

git branch
List all local branches
git branch <name>
Create a new branch
git checkout <name>
Switch to an existing branch
git checkout -b <name>
Create and switch to a new branch
git merge <branch>
Merge a branch into the current one

Remote & Sync

git remote -v
Show connected remote repositories
git push origin <branch>
Push branch to remote
git pull origin <branch>
Pull and merge from remote branch
git fetch
Download remote changes without merging
// Branching Strategy

Branch, Commit, Merge

main
init
v1.0
merge feat
merge fix
feature/login
add form
add auth
tests
hotfix/bug
fix null
tests pass
🌿

Branch Per Feature

Always create a new branch for each feature or bug fix. Never commit directly to main.

✍️

Descriptive Commit Messages

Write clear, present-tense messages: "Add login form validation" not "stuff" or "fix".

🔄

Pull Before You Push

Always pull the latest remote changes before pushing to avoid merge conflicts.

👁️

Review via Pull Requests

Use GitHub Pull Requests so teammates can review code before it enters the main branch.

🚫

Use .gitignore

Keep secrets, node_modules, and build artifacts out of version control with a .gitignore file.

🔖

Tag Your Releases

Use git tag to mark release versions like v1.0.0 for easy reference.
